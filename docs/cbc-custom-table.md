# Creating Custom Tables via Custom Business Configuration (CBC)

**Custom Business Configuration (CBC)** in fact is nothing else than a RAP BO service on top of a customizing table, which is then exposed via an SAP standard generic Fiori Application, like SM30 in SAP GUI. All the RAP artifacts can be generated by the ABAP Repository Object Generator. Which is fast. 

## Prerequisites

- Access to Customizing - Business Configuration Fiori Application. 
  - TCODE: `/n/ui2/flp`â€¯ - Fiori Launchpad 
  - Search Customizing - Business Configuration  

Make sure you are in the correct system.

1. Proceed to ADT - Eclipse for development.
2. Create your Data Elements based on your requirement. 
    <a href="{{ '/media/cbc-custom-table-1.png' | relative_url }}">
    <img src="{{ '/media/cbc-custom-table-1.png' | relative_url }}"
        width="800" />
    </a>
3. Create your custom table using this format: 

```
@EndUserText.label : 'Wire Material' 
@AbapCatalog.enhancement.category : #NOT_EXTENSIBLE 
@AbapCatalog.tableCategory : #TRANSPARENT 
@AbapCatalog.deliveryClass : #C 
@AbapCatalog.dataMaintenance : #ALLOWED 
define table ztc_sapdev_wirem { 

key client            : abap.clnt not null; 
  key wire_material     : zde_sapdev_wire_material not null; 
  configdeprecationcode : config_deprecation_code; 
  last_changed_at       : abp_lastchange_tstmpl; 
  local_last_changed_at : abp_locinst_lastchange_tstmpl; 

} 
```

1. Right click on the **workspace** > **Open others** > **Technical settings**
2. Tick **[Log changes]** 
    <a href="{{ '/media/cbc-custom-table-2.png' | relative_url }}">
    <img src="{{ '/media/cbc-custom-table-2.png' | relative_url }}"
        width="800" />
    </a>
3. Save.
4. Create the Business Configuration Object 
5. Right click on the newly created table > Generate ABAP Repository Objects 
6. Select Maintenance Object > Next > Choose desired package 
    <a href="{{ '/media/cbc-custom-table-3.png' | relative_url }}">
    <img src="{{ '/media/cbc-custom-table-3.png' | relative_url }}"
        width="800" />
    </a>
    <a href="{{ '/media/cbc-custom-table-4.png' | relative_url }}">
    <img src="{{ '/media/cbc-custom-table-4.png' | relative_url }}"
        width="800" />
    </a>
7.  Publish your service binding so that your oData service is accessible 
    <a href="{{ '/media/cbc-custom-table-5.png' | relative_url }}">
    <img src="{{ '/media/cbc-custom-table-5.png' | relative_url }}"
        width="800" />
    </a>
8.  If you cannot  publish it in ADT, go to SAP GUI > [Your Development System] > `/n/IWFND/V4_ADMIN`
9.  Expand Service Groups > Scroll at the end and see if your file is there.
    <a href="{{ '/media/cbc-custom-table-6.png' | relative_url }}">
    <img src="{{ '/media/cbc-custom-table-6.png' | relative_url }}"
        width="800" />
    </a>
10. If not found, click on Publish Service Groups 
    <a href="{{ '/media/cbc-custom-table-7.png' | relative_url }}">
    <img src="{{ '/media/cbc-custom-table-7.png' | relative_url }}"
        width="800" />
    </a>
11. On System Alias, input `LOCAL` then click Get Service Groups 
    <a href="{{ '/media/cbc-custom-table-8.png' | relative_url }}">
    <img src="{{ '/media/cbc-custom-table-8.png' | relative_url }}"
        width="800" />
    </a>
12. Scroll down to the end to see your file, then click **Publish Service Groups** 
    <a href="{{ '/media/cbc-custom-table-9.png' | relative_url }}">
    <img src="{{ '/media/cbc-custom-table-9.png' | relative_url }}"
        width="800" />
    </a>
13. Double check your description to finally publish your service binding. 
14. To check your table, go to the **Fiori Launchpad** > **Search Custom Business Configuration**. You should see your table there.  
    <a href="{{ '/media/cbc-custom-table-10.png' | relative_url }}">
    <img src="{{ '/media/cbc-custom-table-10.png' | relative_url }}"
        width="800" />
    </a>
15. You have now created your custom table via CBC.
